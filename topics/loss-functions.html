The big idea
Different losses reward different things.
A loss function turns how wrong the model is into one number. Squared error punishes big mistakes quadratically; absolute error treats them evenly; cross-entropy punishes confident wrong answers savagely. The gradient of the loss is what actually moves the weights.

Why these choices matter

MSE is smooth, differentiable everywhere, and has a clean closed form for linear regression. But the gradient ∂L/∂ŷ = 2(ŷ − y) grows with the error, so a single outlier can dominate the fit. MAE's gradient is sign(ŷ − y) — constant magnitude, so no one point can dominate. Huber is a compromise: quadratic near 0 (smooth), linear far from 0 (robust).
For a classifier the prediction is a probability p for the true class. BCE = −log p goes to infinity as p → 0, so being both confident and wrong is punished catastrophically. When BCE is composed with a sigmoid/softmax layer, the gradient with respect to the pre-activation logit simplifies to p − y — that clean form is why every modern classifier trains on cross-entropy. Note: the gradient panel shows ∂L/∂p (the raw loss shape), which diverges at the endpoints — the neat p − y only appears after the sigmoid.
When 99% of your examples are "negatives" (e.g. object detection backgrounds), BCE is dominated by easy examples the model already gets right. Focal loss multiplies BCE by (1 − p)^γ: easy examples (p ≈ 1) get down-weighted, hard examples (p ≈ 0) keep their full loss. γ = 0 recovers BCE; γ = 2 is the usual default.
Training never "sees" the loss value directly — only its gradient. The gradient panel shows each loss's ∂L/∂ŷ: MSE gives a linear ramp (bigger error → bigger step), MAE is a step function (flat magnitude → robust), Huber smoothly interpolates. Cross-entropy's gradient through the sigmoid simplifies to p − y, which is what the optimizer actually receives.
