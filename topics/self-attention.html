Self-Attention

Q / K / V matrices, multi-head attention, causal masking, scaling — step by step

1 Input Tokens click to inspect

Each token is one word (or word-piece). The model processes them in parallel, but every token can look at every other. Click one to see what it attends to.

2 Token Embeddings (X)

Each token gets turned into a vector of dmodel numbers. Similar meanings → similar vectors. These are the inputs to the attention layer.

3 Project into Q, K, V Q = XWQ · K = XWK · V = XWV

Three learned matrices turn each token embedding into three different views:
Query (Q): "what am I looking for?" — Key (K): "what do I contain?" — Value (V): "what do I deliver?"

4 Compute Attention Scores — QKT / √dk

For each query, we take the dot product with every key. High score = those two tokens are "aligned." We divide by √dk to keep numbers manageable as the dimension grows (otherwise softmax would saturate).
Without the √dk scaling, softmax would assign ~all probability to one token — gradients die, training fails. This tiny division matters.

5 Apply Causal Mask (GPT-style) lower-triangular

In a decoder (like GPT), position i can only see positions ≤ i — future is masked with −∞. That way, when predicting the next token, the model never "cheats" by looking ahead. Toggle mask in the control panel to see the effect.

6 Softmax (row-wise)

Softmax converts each row of scores into a probability distribution (all positive, sums to 1). Temperature τ controls sharpness: τ ↓ = winner-take-all, τ ↑ = uniform.

7 Weighted Sum of Values — output = A · V

For each token, multiply every value vector by its attention weight, then sum. The result is the attention output for that token — a context-aware representation.

8 Multi-Head Attention each head sees a different "story"

Instead of one attention operation, modern transformers run many in parallel (8, 16, 96, …). Each head has its own WQ, WK, WV. Different heads learn different relationships: grammar, semantic, long-range, positional…

9 Interesting things to try

1. Select a pronoun like "it" or "she" — attention usually peaks on the noun it refers to.
2. Crank τ way up → every token attends uniformly. The network has no meaningful focus.
3. Toggle the causal mask — early tokens can't see later ones, so their attention patterns are very sparse.
4. Pick different heads — the same token can have wildly different attention patterns across heads.
The Big Idea
Attention = "who should I listen to?"
When you read "The cat sat on the mat because it was tired", your brain knows "it" refers to the cat. Self-attention is how transformers do this: every token scores every other token and mixes their values accordingly.

Key Concepts

Think of it as a database lookup. A query is what you're searching for. Keys are the entries' labels. The dot product Q·K tells you how well each key matches the query. The match weights are used to combine the values. Three projections let each role ask different questions about the same input.
If Q and K have variance 1 in each dimension, Q·K has variance dk. Without dividing, the magnitudes grow with dk, pushing softmax into the saturated regime (one value near 1, all others ~0). Gradient flow dies. Dividing by √dk keeps softmax in a useful range.
A single attention head averages across all aspects — so it can only pay attention to one thing at a time. With h heads, each in a lower-dimensional subspace, the model can simultaneously attend to syntactic dependencies, semantic roles, positions, entities, etc. Paper: "Attention is All You Need" (Vaswani et al., 2017).
GPT-style models are causal: a token can only attend to tokens at positions ≤ itself. This is critical for language generation — at training time we can't let the model peek at future tokens.

BERT-style models are bidirectional: every token can attend to every other. BERT is used for understanding (classification, Q&A) but cannot generate text auto-regressively.
Computing QKT for a sequence of length n gives an n×n matrix of scores. Attention is O(n2) in time and memory — the biggest bottleneck for long contexts. Research directions: Flash Attention (better memory layout), Linformer (low-rank), Mamba/SSMs (linear), Ring Attention (distributed), sparse / sliding-window attention (Longformer, Mistral SWA).
The operation you see here runs thousands of times per forward pass of GPT-4, Claude, Gemini, LLaMA, Mistral. Every single layer. With multi-head attention. With causal masks. Same exact math. Scale changes, principle doesn't.
